Web Application


D3M is a collaboration solution online tool built to help technicians, project managers, sales reps, engineers, design network topology, rack and floor plan diagrams all while auto-generating project inventory, quotes, proposals and documentation.
For more information visit d3mnetworks.com

In this project I worked together with the D3M team implementing new features, enhancing exisiting functionality, and redesigning the overall user experience through UX research, prototyping and iterating on solutions.

UI redesign concept

The initial step was evaluating the existing app, identifying issues with the existing user experience, and defining a strategy. During this phase we conducted contextual research, competitive analysis, listen to interviews, and observed users working with the app in real time.

Areas we identified needed improvement;
Onboarding process - the initial step was conversome, users were dropping off before completing the sign up process.
Tools in the app to build and design different diagrams were not intuitive. Users had issues identifying how to use different tools.
Inconsistencies in the UI throughout the app; mix of CSS styles resulting in elements with the same functionality having different looks or different ways of interaction.

The app was initially built without an specific design direction, we did a ‘remapping’ of the app to understand the existing information architecture, and user flow. This exercise also allowed us to identify and reduce long branches, and identify dead ends.

Sketching


I sketch at every step along the UX process. It helps me visualize flows, maps or features and make mental connections I wouldn’t otherwise see.
Sketching layouts and user interfaces is invaluable for getting rapid feedback. It’s also a fast and easy way to get stakeholders’ approval without investing a lot of time into a design.
In an Agile environment, sketching is also helpful as a means of collaborative design.
