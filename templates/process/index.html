Discovery


Define the problem - users and business needs. Specify functional requirements - usually with a PM.



Research


User research, personas, surveys, interviews, ethnography, contextual research, competitive analysis.



Architecture


Synthesize information, build shared understanding - journey maps, mental models, user stories.



Prototyping and Testing


Wireframes. Build interactive prototypes, get early feedback and refine. A/B testing.



Validate and Iterate


Collect feedback - Evaluate test results and determine necessary modifications. Build design system.
